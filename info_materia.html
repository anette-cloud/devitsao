Desarrollo Full Stack

DSH-2502 Junio 2025 · ITS Osorio 14 docentes

Arquitectura completa: front‑end accesible + back‑end con patrones. Angular, Firebase, REST, CI/CD y control de versiones.

Diseñar, implementar y desplegar aplicaciones web escalables, responsivas y mantenibles.

Competencia específica

Diseñar e implementar interfaces web accesibles, responsivas y usables con HTML5, CSS3, Bootstrap, Angular, además de patrones de diseño back‑end (Singleton, Factory, Adapter, Observer, State…) para arquitecturas sólidas y escalables.

Competencias previas

  • POO + funcional: clases, herencia, encapsulamiento
  • Diagramas UML (clases / secuencia)
  • Algoritmos, arreglos, listas, depuración
  • Bases HTML, CSS, JS y modelo cliente‑servidor
  • CRUD básico en BD relacionales / no relacionales

Intención didáctica

Comprender y aplicar de forma integral herramientas, patrones y prácticas (front/back, control de versiones, CI/CD) para construir y mantener aplicaciones web completas, escalables y accesibles.

Temario · 4 ejes

1. Fundamentos Full Stack

Front-End / Back-End, cliente-servidor, hosting, Node.js, NPM, Git, GitHub Actions, CI/CD.

Arquitectura capas + control versiones

2. Interfaces accesibles

HTML5 semántico, CSS3, media queries, Bootstrap/Tailwind, WCAG, diseño responsivo.

Auditoría Lighthouse / WAVE

3. Patrones Back-End

Singleton, Factory, Adapter, Decorator, Composite, Observer, State. Conexión BD, máquina de estados.

Creacionales / estructurales / comportamiento

4. Front-End con Angular

CLI, componentes, servicios, JSON, HTTP, async/await, Firebase, formularios reactivos, rutas, 404.

Angular + Firebase BaaS

Actividades clave

  • Infografía front-end vs back-end vs full stack
  • Diagrama cliente-servidor / OSI + hosting con GitHub Pages
  • Instalación Node, NPM, Git, VSC; repo con ramas
  • Página semántica + media queries; Bootstrap/Tailwind cards, navbar
  • Patrones: Singleton conexión, Factory user, Adapter API, Decorator notas
  • Observer notificaciones / State para orden
  • Angular: 3 componentes + servicio, HTTP (GET,POST,PUT,DELETE)
  • Formulario reactivo con validaciones, ruteo dinámico, Firebase Auth
Prácticas (11 en total): factory method, adapter, decorator, angular cli, jsonplaceholder, reactive forms, menú mobile, accesibilidad ARIA, etc.
